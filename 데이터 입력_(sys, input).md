# 데이터 입력
<p>코딩 테스트에서 데이터를 입력 받을 때 보통 사용하는 방법은 input과 sys.stdin.readline 2가지이다.</p>
<p>보통 코딩 테스트의 데이터는 현재 연결된 표준 입력(STDIN) 장치에서 읽어온다</p>


# input
<p>input(prompt, /)</p>
- prompt 인자가 입력되면, 표준 출력(화면)에 줄바꿈 없이 내용이 작성됨
- 그 후 함수는 입력에서 한 줄을 읽어서, 문자열로 변환하여 반환(줄바꿈 문자는 제거)
- 만약 파일이 끝나서 입력값이 더 이상 없으면 EOFError 발생
  
### prompt인자
prompt 인자는 필수는 아님
- prompt가 있을 때
  - 대화형 환경
  - 목적 : 터미널이나 로컬 환경에서 사용자와 상호작용할 때 적합
  - ex) input('값을 입력하세요: ')
    - 작동 : input()함수는 먼저 표준 출력에 '값을 입력하세요 : '를 프롬프트로 출력하고 사용자가 값을 입력하길 기다림
- prompt가 없을 때
  - 코딩테스트 환경
  - 목적 : 안내 메세지가 필요 없이 바로 데이터 읽을 때 적합
  - ex) N = int(input())
    - 작동 : 표준 입력 스트림에서 데이터가 입력되기를 기다림 (코딩 테스트 서버는 이미 데이터 파일이 표준 입력 스트림으로 연결되어 있음)


# sys.stdin.readline
### stdin
- sys 라이브러리(모듈) 내 객체(object) / 함수 아님
  - 표준 입력 장치와의 연결 통로를 나타내는 객체 (file처럼)
 
### readline
- sys.stdin 객체에 붙어 있는 기능 - method




## 비교 (비교적)
- 사용 : data = input()
- 속도 : sys(빠름) - input(느림)
- 입력 데이터 : sys(파일 전체) - input(한줄)
- input은 대화형으로 생성되어 내부 프롬프트 출력 관리 로직이 있음. sys는 없이 읽어오기에만 집중
- 한 줄 씩 데이터를 입력 받을 때 input이 적합  
  - 대화형, 프롬프트형
- 대량의 데이터를 입력 받을 때 sys가 적합
  - 대량의 파일, 코딩테스트
### input
- 라이브러리 : 내장함수
- 속도 : 비교적 느림
- 데이터를 한 줄 씩 입력 받음
- 줄바꿈 : 자동으로 제거하고 반환
  - ex. 입력값이 "123\n"면 123 반환
 
### sys.stdin.readline
- 라이브러리 : sys
- 속도 : 비교적 빠름
- 데이터를 한 번에 입력 받음
- 줄바꿈 :줄 끝의 **\n (줄바꿈 문자)**을 포함하는 등 추가 처리 필요
  - ex. 입력값이 "123\n" 면 123\n 반환
- 사용 : import sys (라이브러리 필요)
    data = sys.stdin.readline()
